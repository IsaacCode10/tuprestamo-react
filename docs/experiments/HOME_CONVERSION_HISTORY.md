# Historial de Iteraciones Home Conversion

## Objetivo

Este documento registra la evolucion de la home de Tu Prestamo como activo de conversion.

La idea es documentar:

- que version estuvo en produccion
- que problema intentaba resolver
- que cambios se hicieron
- que metricas tuvo antes y despues
- que aprendimos

Esto sirve para:

- comparar iteraciones con evidencia
- evitar decidir por memoria o intuicion
- construir aprendizaje acumulado
- explicar la evolucion del producto a futuros colaboradores
- fundamentar decisiones de growth, producto y UX

---

## Versiones

### V0 - Baseline Home Generalista

- Documento: [v0_baseline.md](./home/v0_baseline.md)
- Estado: Documentada
- Objetivo historico: presentar el negocio de forma general para prestatarios e inversionistas
- Diagnostico posterior: cuello de botella fuerte antes del formulario

### V1 - Foco Prestatario

- Documento: [v1_foco_prestatario.md](./home/v1_foco_prestatario.md)
- Estado: Planificada
- Objetivo: aumentar conversion de visitantes a intencion real de prestatario
- Hipotesis: una home mas enfocada en tenedores de tarjeta aumentara la tasa de paso hacia el formulario

---

## Metricas canonicas para comparar versiones

Estas son las metricas base que se deben usar para comparar una version contra otra:

- Unique Page Views
- Calculated Loan Result
- Viewed Loan Application Form
- Started Loan Application
- Submitted Loan Application
- Viewed Borrower Dashboard

Cuando aplique:

- Clicked Auditor CTA

---

## Regla de documentacion

Cada nueva iteracion de home debe registrar:

1. fecha de inicio
2. fecha de deploy
3. problema que intenta resolver
4. hipotesis
5. cambios aprobados
6. cambios descartados
7. metricas baseline
8. metricas posteriores
9. aprendizajes
10. siguiente accion
